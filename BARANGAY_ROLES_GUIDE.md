## ğŸ“‹ Document Request Workflow

Here's how different roles work together:

### Example: Resident requests Barangay Clearance

**Step 1: Request Submission**
- **Resident**: Logs in, fills out request form
- System: Creates request with status "Pending"

**Step 2: Processing**
- **Secretary**: Reviews request, prepares document
- Secretary: Changes status to "Processing"
- Secretary: May ask resident for additional info

**Step 3: Approval**
- **Captain**: Reviews the prepared document
- Captain: Approves or rejects
- If approved: Status changes to "Approved"
- If rejected: Status changes to "Rejected" with reason

**Step 4: Payment**
- **Resident**: Pays the fee (â‚±50 for clearance)
- **Treasurer**: Confirms payment received
- Treasurer: Marks request as "Paid"
- Treasurer: Issues official receipt

**Step 5: Release**
- **Secretary**: Releases the document to resident
- Secretary: Changes status to "Released"
- Resident: Can now download/print the document

## ğŸ’¼ Real-World Scenarios

### Scenario 1: New Resident Registration

**Who does what**:
1. **Resident**: Registers online via registration form
2. **Secretary**: Verifies information, adds to system
3. **Admin**: Reviews and approves if needed
4. **Resident**: Can now request documents

### Scenario 2: Barangay Clearance Request

**Who does what**:
1. **Resident**: Submits request online
2. **Secretary**: Reviews, prepares clearance document
3. **Captain**: Reviews and approves
4. **Resident**: Pays â‚±50 fee
5. **Treasurer**: Confirms payment
6. **Secretary**: Releases document to resident

### Scenario 3: Certificate of Indigency (Free)

**Who does what**:
1. **Resident**: Submits request with supporting documents
2. **Secretary**: Verifies indigency status
3. **Captain**: Reviews and approves (or rejects)
4. **Secretary**: Releases document (no payment needed)

### Scenario 4: Adding New Staff

**Who does what**:
1. **Admin**: Logs in, goes to Users page
2. **Admin**: Clicks "Add User"
3. **Admin**: Fills in: Name, Email, Role, Password
4. **Admin**: Saves user
5. **Admin**: Shares credentials with new staff member
6. **New Staff**: Logs in, changes password

## Summary

- **Super Admin**: Manages entire system (all barangays)
- **Admin**: Manages one barangay (full control)
- **Captain**: Approves documents (final decision maker)
- **Secretary**: Processes documents (day-to-day operations)
- **Treasurer**: Manages payments (financial transactions)
- **Resident**: Requests documents (service user)

Each role has specific responsibilities and access levels to ensure efficient and secure barangay operations.
